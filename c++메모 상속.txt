---------상속----------------------------

다른 클래스의 특성들을 내려받음
-base class 부모
-파생 derived class 자식

자식 클래스는 다음을 포함
-부모의 멤버 변수/함수
-부모의 생성자와 소멸자
-자식이 고유로 추가한 멤버 변수/함수

자식클래스가 나중에 생성되고 먼저 소멸됨
(LIFO)

-------다중 상속----------------------------------
문제점
상속받은 여러 부모 클래스에 같은 이름의 멤버가 존재하면?
하나의 클래스를 두번 이상 상속 받을 수도.
프로그래밍이 복잡해져서 실용성이 낮음

부모1의 부모클래스인 A호출,
부모2의 부모클래스인A호출
->클래스A가 두번 호출됨

가급적이면 자바의 인터페이스 형태로만 다중상속을 사용할것

-----------다형성---(polymorphism)---------------
하나가 다양한 형태의 함수를 가지는 것
(상황에 따라 다르게 적용)

동일한 함수들이 다른 행동을 하는 것
ex) 호랑이.짖다() //어흥
        개.짖다()  //멍멍

부모클래스가 아닌 자식클래스의 소멸자 함수에도 virtual을 써줌
(자식클래스가 나중에 부모클래스가 될 수 있기 때문에)

--------순수 가상함수------
java의 추상메소드
-실체(정의)가 없는 함수
-자식클래스에서 overriding을 하자
-virtual 함수시그니쳐 = 0 //정의는 자식클래스에서 하겠다는 의미
	ex)void eat(void)
-추상메소드가 하나라도 있는 클래스(추상클래스)는
   instance를 생성할 수 없음(실체가 없으니까)
